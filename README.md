# senijuense
多功能多协议工业网关，覆盖 GPIO / I2C / SPI 传感器采集、RS485（Modbus）、SocketCAN 等模块开发。

Embedded IoT Gateway System
1. 项目简介

本项目基于 Linux 平台，实现了一套轻量级嵌入式物联网数据网关系统。系统具备数据采集、本地存储、图形化显示以及云端上传等功能。

整体采用多线程架构，并通过自定义邮箱机制实现线程间通信，从而达到模块解耦、提高系统稳定性与扩展性的目的。

适用于工业环境监测、边缘计算节点及嵌入式数据采集等场景。

2. 系统架构

系统采用多线程设计，各功能模块运行在独立线程中，通过邮箱机制进行数据交互。

主要线程包括：

数据采集线程：负责采集或模拟传感器数据
存储线程：负责数据持久化（SQLite）
MQTT线程：负责数据上云
显示线程：负责本地图形界面显示

线程之间不直接耦合，所有数据通过邮箱进行分发。

3. 核心功能
3.1 多线程管理机制
提供线程注册接口
支持多任务并发执行
统一线程生命周期管理
3.2 邮箱通信机制

系统实现了一套基于链表的线程邮箱机制，用于线程间通信，核心特性包括：

基于互斥锁（pthread_mutex）保证线程安全
基于条件变量（pthread_cond）实现阻塞与唤醒
支持一对多消息分发

该机制避免了轮询带来的 CPU 开销，提高系统效率。

3.3 数据存储模块

基于 SQLite 实现本地数据存储，支持：

自动建表
数据插入
按时间范围查询
历史数据清理

同时通过互斥锁保证多线程访问安全。

3.4 MQTT 通信模块

基于 Paho MQTT C 客户端实现数据上传，主要功能：

数据发布（JSON 格式）
连接管理与回调处理
QoS 控制
3.5 Framebuffer 显示模块

基于 Linux framebuffer 实现图形显示，无需依赖 GUI 框架，支持：

BMP 图片绘制
ASCII 字符显示
汉字点阵显示
实时数据刷新
3.6 日志系统

提供统一日志接口，支持：

多级日志输出（ERROR / WARN / INFO / DEBUG）
按日期生成日志文件
历史日志清理

5. 数据流程

系统数据流如下：

数据采集线程生成数据
通过邮箱分发至各功能线程
存储线程写入数据库
MQTT线程上传云端
显示线程更新界面
6. 编译与运行
6.1 依赖环境
Linux 操作系统
gcc 或交叉编译工具链
pthread
sqlite3
paho-mqtt C 库
6.2 编译
make

如需交叉编译，请修改 Makefile 中的编译器配置。

6.3 运行
./app

程序运行后，输入 q 并回车可退出。

7. 数据来源说明

当前系统使用随机数模拟传感器数据，后续可替换为真实硬件采集，例如：

温湿度传感器（DHT11 等）
电压/电流采集模块
I2C / SPI 外设
